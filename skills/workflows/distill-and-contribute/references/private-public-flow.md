# Private / Public Flow

## 默认模式

默认使用 `private-first`：

- 先写入 `workspace/private/*`
- 先形成 patch proposal
- 先让用户确认

## 两种模式

### private-first

适用于：

- 个人经验
- 案例复盘
- 暂未去敏感化内容
- 尚未确认是否公开的草稿

### public-candidate

适用于：

- 已明确可公开
- 已去敏感化
- 已能补齐来源和边界
- 准备进入贡献流程

## promotion path

1. private draft
2. public candidate
3. merge to public

任何时候都不要自动跳过用户确认。
