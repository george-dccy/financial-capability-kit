# 开场框架

## 框架一：信任-时机矩阵

横轴：信任程度（低→高）
纵轴：时间宽裕度（紧→宽）

四个象限对应不同开场策略：

```
                    低信任                    高信任
            ┌─────────────────────┬─────────────────────┐
     时间紧 │   策略A：价值先示    │  策略B：快速切入     │
            │  30秒抛价值钩子      │  直接进主题          │
            │  不做深度提问        │  用确认代替探索      │
            ├─────────────────────┼─────────────────────┤
     时间宽 │   策略C：信任建立    │  策略D：开放探索     │
            │  先聊共性问题        │  用框架引导          │
            │  观察反应再调整      │  深度共建议程        │
            └─────────────────────┴─────────────────────┘
```

## 框架二：SPIN 改编版（银行对公场景）

SPIN 是顾问式销售的经典框架（Situation → Problem → Implication → Need-payoff）。
银行对公场景中，做以下改编：

### Situation 现状问题（谨慎使用）

- 适合：已有关系、客户愿意分享
- 不适合：首次接触、冷淡客户
- 银行针对性：优先问经营动态、交易习惯，而非内部流程

**改编要点**：问"最近"不问"一直"，问"动态"不问"静态"

### Problem 困难问题（核心武器）

- 适合：所有类型客户
- 关键：问"卡点"不问"需求"
- 银行针对性：问资金周转、账期、回款、融资成本等真实痛点

### Implication 影响问题（谨慎定量）

- 适合：客户有明显损失场景
- 风险：银行场景中不宜直接算客户损失
- 银行针对性：用"这对您的业务连续性意味着什么"代替具体金额

### Need-payoff 目标问题（转化利器）

- 适合：高信任、有决策权的客户
- 关键：让客户说结果，不是你给结论
- 银行针对性：问"理想状态"而非"满意阈值"

## 框架三：客户状态快速评估四问

开场后 2 分钟内，通过以下信号评估客户状态：

1. **响应速度**：问题提出后几秒内回应？（快=感兴趣，慢=需要思考或不想聊）
2. **信息密度**：客户回答时提供多少细节？（细节多=配合度高）
3. **情绪方向**：语气是中性、防御还是开放？（开放=可深入）
4. **动作信号**：客户是否有记录、追问、转发给同事等动作？（有=有实际意义）

## 框架四：开场策略选择流程

```
第一步：判断信任基础
  ├── 首次接触 → 进入冷开场路径（策略A或C）
  └── 已有关系 → 进入热开场路径（策略B或D）

第二步：判断时间窗口
  ├── 时间紧 → 优先策略A或B（快）
  └── 时间宽 → 可选策略C或D（深）

第三步：选择提问方向
  ├── 冷开场 → 先抛价值钩子，再跟影响/目标问题
  └── 热开场 → 直接进现状+困难问题

第四步：预判配合度
  └── 根据前三步综合评分，决定提问深度
```
