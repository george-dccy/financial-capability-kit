# 客户拜访开场框架

## Scope

这是一个 reference skill。
它不提供具体话术，而是提供开场时机的判断框架、信任建立策略选择、以及客户状态快速评估方法。

## When To Use

- 准备拜访客户，不知道先聊什么
- 需要判断用哪种开场策略
- 想快速评估客户当前状态和配合意愿
- 需要设计开场提问方向，而不是照读话术

## Core Output Rule

每次开场至少明确：

1. 选择哪种开场策略（冷淡/常规/高信任）
2. 第一个问题往哪个方向走（现状/困难/影响/目标）
3. 预判客户可能的配合度
4. 明确本 skill 与 opening-playbook 的使用边界

## Not For

- 提供具体话术文本
- 替代 action skill 的完整场景编排
- 首次拜访后的持续跟进设计

## 判断框架

### 框架一：信任-时机矩阵

横轴：信任程度（低→高）
纵轴：时间宽裕度（紧→宽）

四个象限对应不同开场策略：

```
                    低信任                    高信任
            ┌─────────────────────┬─────────────────────┐
     时间紧 │   策略A：价值先示    │  策略B：快速切入     │
            │  30秒抛价值钩子      │  直接进主题          │
            │  不做深度提问        │  用确认代替探索      │
            ├─────────────────────┼─────────────────────┤
     时间宽 │   策略C：信任建立    │  策略D：开放探索     │
            │  先聊共性问题        │  用框架引导          │
            │  观察反应再调整      │  深度共建议程        │
            └─────────────────────┴─────────────────────┘
```

### 框架二：SPIN 改编版（银行对公场景）

SPIN 是顾问式销售的经典框架（Situation → Problem → Implication → Need-payoff）。
银行对公场景中，做以下改编：

**Situation 现状问题（谨慎使用）**
- 适合：已有关系、客户愿意分享
- 不适合：首次接触、冷淡客户
- 银行针对性：优先问经营动态、交易习惯，而非内部流程
- 改编要点：问"最近"不问"一直"，问"动态"不问"静态"

**Problem 困难问题（核心武器）**
- 适合：所有类型客户
- 关键：问"卡点"不问"需求"
- 银行针对性：问资金周转、账期、回款、融资成本等真实痛点

**Implication 影响问题（谨慎定量）**
- 适合：客户有明显损失场景
- 风险：银行场景中不宜直接算客户损失
- 银行针对性：用"这对您的业务连续性意味着什么"代替具体金额

**Need-payoff 目标问题（转化利器）**
- 适合：高信任、有决策权的客户
- 关键：让客户说结果，不是你给结论
- 银行针对性：问"理想状态"而非"满意阈值"

### 框架三：客户状态快速评估四问

开场后 2 分钟内，通过以下信号评估客户状态：

1. **响应速度**：问题提出后几秒内回应？（快=感兴趣，慢=需要思考或不想聊）
2. **信息密度**：客户回答时提供多少细节？（细节多=配合度高）
3. **情绪方向**：语气是中性、防御还是开放？（开放=可深入）
4. **动作信号**：客户是否有记录、追问、转发给同事等动作？（有=有实际意义）

### 框架四：开场策略选择流程

```
第一步：判断信任基础
  ├── 首次接触 → 进入冷开场路径（策略A或C）
  └── 已有关系 → 进入热开场路径（策略B或D）

第二步：判断时间窗口
  ├── 时间紧 → 优先策略A或B（快）
  └── 时间宽 → 可选策略C或D（深）

第三步：选择提问方向
  ├── 冷开场 → 先抛价值钩子，再跟影响/目标问题
  └── 热开场 → 直接进现状+困难问题

第四步：预判配合度
  └── 根据前三步综合评分，决定提问深度
```

## 示例

### 示例一：首次接触 + 时间紧（策略A）

场景：第一次拜访某制造业客户，财务总监只给了15分钟。

判断过程：
1. 首次接触 + 低信任 → 策略A（价值先示）
2. 时间紧 → 快进快出
3. 不做现状问题，直接抛影响问题

框架输出：
- 选择策略：价值先示+快切主题
- 第一个问题方向：影响问题（"如果这个问题持续，对您近期业务目标意味着什么"）
- 预判：客户配合度中等，不深入展开

### 示例二：已有关系 + 时间宽（策略D）

场景：老客户年度拜访，总经理安排了1小时。

判断过程：
1. 已有关系 + 高信任 → 策略D（开放探索）
2. 时间宽裕 → 可以深挖
3. 从困难问题入手，引导到目标问题

框架输出：
- 选择策略：开放探索+共建议题
- 第一个问题方向：困难问题（"今年以来，哪个环节最费您的时间"）
- 预判：客户配合度高，可以展开四类问题

### 示例三：评估后临时调整

场景：开场时预判是策略B（快速切入），但客户响应很慢。

调整过程：
1. 状态评估发现：客户回应需要10秒以上，信息密度低
2. 判断：这不是时间紧的问题，而是信任不足或话题不感兴趣
3. 调整：从策略B切换到策略C（信任建立），降速提问

## 常见误区

### 误区一：上来就推产品

表现：开场前3分钟讲完所有产品功能，等客户反应。
为什么错：破坏了信任建立窗口，客户一旦觉得被推销，后续所有问题都会被当成营销。
判断框架：如果开场用了"我们产品可以……"开头，即触发此误区。

### 误区二：问太多现状问题

表现：像查户口一样连续问5个以上的现状问题。
为什么错：在低信任或时间紧的场景下，现状问题会迅速消耗客户的耐心和配合意愿。
判断框架：首次接触现状问题超过2个即触发；已有关系超过4个且无困难/目标问题跟进即触发。

### 误区三：过早谈费率、额度、审批结论

表现：开场5分钟内开始讨论具体价格或审批结果。
为什么错：银行对公业务中，过早谈结论是最大的信任破坏行为。
边界：可以谈"大概方向"，但不能谈具体数字和承诺。

### 误区四：不做会后动作锁定

表现：聊完就说"有需要随时联系我"，没有明确下一步。
为什么错：没有锁定动作的拜访，只是信息交换，不是关系推进。
