# 示例

## 示例一：首次接触 + 时间紧（策略A）

**场景**：第一次拜访某制造业客户，财务总监只给了15分钟。

**判断过程**：
1. 首次接触 + 低信任 → 策略A（价值先示）
2. 时间紧 → 快进快出
3. 不做现状问题，直接抛影响问题

**框架输出**：
- 选择策略：价值先示+快切主题
- 第一个问题方向：影响问题（"如果这个问题持续，对您近期业务目标意味着什么"）
- 预判：客户配合度中等，不深入展开

## 示例二：已有关系 + 时间宽（策略D）

**场景**：老客户年度拜访，总经理安排了1小时。

**判断过程**：
1. 已有关系 + 高信任 → 策略D（开放探索）
2. 时间宽裕 → 可以深挖
3. 从困难问题入手，引导到目标问题

**框架输出**：
- 选择策略：开放探索+共建议题
- 第一个问题方向：困难问题（"今年以来，哪个环节最费您的时间"）
- 预判：客户配合度高，可以展开四类问题

## 示例三：冷淡客户尝试破冰（策略C）

**场景**：客户多次表示"没时间"，但没有明确拒绝。

**判断过程**：
1. 低信任 + 多次推脱 → 策略C（信任建立优先）
2. 时间不确定 → 先试探
3. 用共性问题降低防御，不直接推产品

**框架输出**：
- 选择策略：信任建立
- 第一个问题：共性问题（行业动态、近期政策），不是业务问题
- 预判：客户配合度低，需要2-3次才能判断真实状态
- 注意：如果连续两次都是推脱信号，考虑暂停主动接触

## 示例四：评估后临时调整

**场景**：开场时预判是策略B（快速切入），但客户响应很慢。

**调整过程**：
1. 状态评估发现：客户回应需要10秒以上，信息密度低
2. 判断：这不是时间紧的问题，而是信任不足或话题不感兴趣
3. 调整：从策略B切换到策略C（信任建立），降速提问

**框架输出**：
- 原策略：策略B快切主题
- 评估信号：响应慢+信息密度低
- 调整后策略：策略C，先用共性问题试探
- 提问转向：先问"最近行业有什么变化"，不直接进业务问题
