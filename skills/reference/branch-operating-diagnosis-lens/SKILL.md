# 经营分析视角 Skill

## Scope

这是一个 reference skill。
它解决的不是"替你写经营报告"，而是当你拿到一组经营数据、目标和自我判断时，先知道该怎么看、先分什么层、哪些地方要继续追问。

## When To Use

- 需要先判断分支行、网点或团队经营结果是好还是不好
- 需要区分问题更像结果问题、结构问题、过程问题还是外部环境问题
- 需要为后续经营建议、汇报简报或动作排序提供稳定判断框架
- 需要在"数据很多但不知道先看什么"时快速收束视角

## Core Output Rule

每次进入经营分析，至少先明确：

1. **结果判断**：当前结果到底是亮点、持平还是短板
2. **结构判断**：结果背后是客户结构、产品结构、区域结构还是过程动作问题
3. **变化判断**：与目标、上期、去年同期相比，变化到底来自哪里
4. **归因判断**：是内生经营能力问题，还是外部环境和阶段性因素影响
5. **下一步判断**：当前还缺哪些关键补数，哪些结论可以先说，哪些要保留

## Not For

- 直接给出内部经营考核结论
- 替代具体的经营分析 action skill
- 在没有口径一致的数据时硬写"结论先行"的经营报告
- 把公开行业常识冒充为当前机构的真实经营状态

## Quality Gate

- 是否优先回答"结果好不好、结构好不好、变化来自哪里"
- 是否把判断框架和执行脚本分开
- 是否避免落入流水账式报数
- 是否明确写出待补数项和边界

## 判断框架

### 1. 先看结果，不先解释

经营分析的第一步不是解释原因，而是先判断：当前目标完成得怎么样、当前主要结果指标是亮点还是短板、这种变化是阶段波动还是持续趋势。如果"结果判断"本身都没站稳，后面的归因通常会跑偏。

### 2. 经营分析的最小路径

推荐先按四层看：

1. **结果** -- 看目标、增量、同比、环比、绝对值
2. **结构** -- 看客户结构、产品结构、区域结构、客群结构
3. **原因** -- 看是过程动作、资源配置、资产质量还是外部环境因素
4. **动作** -- 看哪些问题值得立刻推进，哪些问题要先补数再判断

简单说就是：`结果 -> 结构 -> 原因 -> 动作`

### 3. 先抓哪几类核心指标

公开经营材料里，银行常用来观察经营质量的指标通常包括：资产负债规模和增速、存款和贷款增长及结构、净息差、净利息收入、非息收入、成本收入比、不良贷款率、拨备覆盖率等资产质量指标、普惠小微涉农等重点领域投放情况。

对分支行和网点的轻量经营分析，优先可以抓：存款、贷款、中收、利润或营收贡献、AUM、客户数/活跃客户、新户/流失。

### 4. 结果问题和结构问题怎么区分

- **更像结果问题**：总量没达标、增速明显掉队、与目标或同业相比差距大
- **更像结构问题**：总量还在但高价值客户占比恶化、增长来自低质量或不可持续部分、客户和产品过度集中
- **更像过程问题**：触达、拜访、跟进、协同、转化节奏弱，管理动作和资源投入跟不上
- **更像外部环境问题**：区域产业波动、政策变化、行业风险暴露、同业竞争挤压

### 5. 亮点和短板不要只看绝对值

经营分析里最常见的误区是只看"多了还是少了"。至少还要一起看：同比、环比、与目标比、与结构比、与质量比。比如贷款增长是亮点，但如果不良抬升、集中度升高，就不能只当亮点。

### 6. 先给初判，再列缺口

轻量经营分析最稳的写法不是"我已经判断完了"，而是：当前初判 / 支撑依据 / 还缺哪些关键补数 / 哪些结论暂不宜下。

### 7. 公开依据映射

- 2025 年一季度银行业保险业主要监管指标继续使用总资产、贷款投放、普惠小微、涉农、不良贷款率等指标观察行业整体运行
- 大型银行公开业绩说明中，通常同时看贷款和存款增长、净息差、成本收入比、非息收入占比、资产质量和结构优化
- 因此分支行经营分析的公开可复用框架，也应兼顾"结果、结构、质量、成本、风险、增长持续性"几个视角

## 示例

### 示例 1：先做结果判断

输入：存款同比增长、贷款增量不错、中收承压，自评认为"整体还可以"。
推荐输出方向：先判断亮点和短板分别是什么，不要用"整体还可以"直接替代经营结论，先把"规模增长"和"收入承压"拆开看。

### 示例 2：先拆结构，再讲原因

不好的写法："贷款没上去，主要因为市场不好。"
更稳的写法："当前贷款结果承压，但还需要继续拆客户结构、产品结构和过程动作，才能判断是外部环境主导，还是资源投入和推进节奏问题。"

### 示例 3：待补数怎么写

推荐写法：当前亮点 / 当前短板 / 初步归因 / 还缺的关键补数 / 暂不宜下结论的点。

## 常见误区

1. **只报数字，不做判断** -- 数字很多不等于经营分析完整。经营分析首先是判断，不是报表转述。
2. **只看结果，不看结构** -- 总量增长不代表结构健康，短板也不一定只在结果层。
3. **先讲原因，再看证据** -- 很多经营分析一开始就把原因说死，后面才补数字，这很容易带偏。
4. **没有待补数项** -- 轻量经营分析最怕假装完整。缺关键数据时，应明确写出还缺什么。
