# 线上授信准入策略建议 Skill

## Scope

这是一个 reference skill。
它聚焦"准入策略怎么想、怎么表达、怎么留边界"，用于线上授信自动化审批业务方案中的准入策略章节。

## When To Use

- 需要为线上授信自动化审批业务方案设计准入策略章节
- 已有业务模式和场景信息，但不确定从哪些维度判断准入
- 需要区分"可给建议的框架"和"必须内部拍板的政策"
- 需要为 action skill 提供稳定的准入策略判断框架

## Core Output Rule

每次给准入策略建议时，至少要分清楚：

1. **业务模式**：这是什么样的对公或产业场景融资
2. **准入对象**：谁是核心企业、平台、经销商、供应商或借款主体
3. **还款来源**：靠真实交易、经营现金流、应收账款、回款归集还是其他来源
4. **数据可核验性**：哪些关键数据真实、连续、可交叉验证
5. **风控与管理闭环**：反欺诈、反洗钱、资金用途、放款审核、贷后管理是否有抓手

## Not For

- 直接设定准入阈值、评分分数线或命中规则
- 替代内部风控模型、授信政策或审批意见
- 在没有场景、数据和还款来源信息时硬写"准入标准"

## Quality Gate

- 是否按业务模式、场景、主体、还款来源、数据、风控、风险缓释几个维度组织
- 是否把"建议框架"和"内部政策"分开
- 是否适合直接嵌入"授信方案"章节
- 是否明确写了待补资料和待人工确认项

## 判断框架

### 1. 先判断是不是适合做线上化准入

优先问五件事：

1. 交易或业务场景是否真实、稳定、可复制
2. 核心企业或平台是否能对交易真实性和履约链条形成稳定支撑
3. 借款主体和还款来源是否清楚
4. 数据是否连续、可验证、可留痕
5. 放款、回款、贷后管理是否能形成闭环

如果这五件事里有两三件都讲不清，准入策略章节应先以"补数和边界提示"为主，而不是硬写详细准入条件。

### 2. 准入策略建议的七个维度

**A. 业务模式与场景类型** -- 是围绕真实贸易、订单、应收、库存、平台交易还是渠道分销；场景价值是否来自真实经营协同而非单纯流量导流；是否具备可持续性和可复制性。

**B. 核心企业或平台支撑力** -- 是否在交易链条中有稳定控制力；能否提供真实交易、结算、发票、履约等关键信号；是否有持续合作基础和历史表现。

**C. 借款主体与还款来源** -- 借款主体身份是否清楚；还款来源是否主要来自真实经营现金流或可归集回款；是否存在还款来源弱、链条过长、依赖单点的情形。

**D. 数据覆盖与核验能力** -- 是否能拿到身份、交易、结算、合同、发票、物流、税务等关键数据中的合理组合；是否能做交叉验证而非只依赖单一平台自报；数据能否支持准入、反欺诈、放款审核和贷后监测。

**E. 风险缓释与资金控制** -- 是否有回款归集、应收转让、保证金、担保、差额补足、资金闭环等风险缓释抓手；风险缓释是实质抓手还是名义安排；能否和场景、交易、回款链条形成联动。

**F. 风控与管理闭环** -- 是否具备身份核验、反欺诈、反洗钱、用途管理、放款审核、贷后预警和催收处置安排；是否存在只重获客和放款、不重贷后和消保的短板。

**G. 消保与合规边界** -- 合作方是否承担清晰职责；咨询、投诉、信息披露、客户授权等环节是否有安排；是否存在对消费者或小微客群不透明、不适当的风险。

### 3. 常见对公场景的观察重点

- **核心企业上下游场景**：重点看核心企业稳定性、上下游真实性、回款链条和应收确认机制
- **平台交易场景**：重点看平台控制力、交易真实性、数据连续性、商户筛选能力和贷后跟踪能力
- **渠道经销或分销场景**：重点看渠道层级、真实销售回款、库存和应收周转、核心品牌约束力
- **物流或仓储场景**：重点看货物流转、仓单或物流信息真实性、回款节点和资金控制安排

### 4. 建议写法

更稳的写法不是直接给阈值，而是：先定义准入对象范围，再说明建议考察维度，再说明哪些资料和数据支撑这些维度，最后说明具体阈值和命中规则待内部政策确认。

### 5. 公开依据映射

- 《商业银行互联网贷款管理暂行办法》强调线上贷款核心环节、风险数据和风险模型要求
- 2025 年互联网助贷业务管理通知强调总行集中管理、平台准入、自主风控、消保责任和关键风险指标管理
- 《个人贷款管理办法》等信贷管理规则强调审慎经营、用途管理、贷前贷后和风险控制要求
- 商业银行官方供应链金融介绍通常强调真实交易背景、核心企业支撑、全流程线上化、回款和交易闭环、风险可控

## 示例

### 示例 1：核心企业上下游场景

推荐表达："准入策略建议围绕核心企业稳定合作基础、上下游真实交易关系、应收或订单可确认性、回款链条可归集性和贷后跟踪抓手进行设计。具体准入阈值、集中度限制和例外情形待内部政策确认。"

### 示例 2：平台交易场景

推荐表达："准入策略建议优先覆盖商户基本经营稳定性、平台交易连续性、订单和结算数据真实性、异常交易识别能力、客户授权和贷后监测安排。对于只具备流量导流、缺少持续交易和结算信息的场景，不建议设置为标准化自动准入场景。"

### 示例 3：补数不足时

推荐输出：当前可建议的准入维度 / 当前缺少的关键资料 / 暂不宜细化的内容 / 待人工确认项。

## 常见误区

1. **把准入策略写成内部政策** -- 公开 skill 适合提供框架，不适合直接给具体阈值和命中规则。
2. **只看流量，不看真实交易** -- 没有真实交易、回款和履约链条支撑的场景，准入策略容易失真。
3. **只看核心企业名气，不看实际控制力** -- 核心企业知名不等于对上下游交易、回款和风险处置有真实抓手。
4. **不写待人工确认项** -- 这样会让方案看起来很完整，但真实落地时最容易出风险。
