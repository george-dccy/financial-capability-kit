# Landing Directions

## 1. 落地输出类型

根据用户意图选择 1-3 个方向，不要一次性堆满。

| 用户意图 | 推荐产出 |
|---|---|
| 我怎么看 | 一句话判断、影响路径、关键变量 |
| 怎么跟客户讲 | 客户版解释、提问清单、切入口 |
| 怎么推进业务 | 目标客户、触达顺序、下一步动作 |
| 怎么给领导汇报 | 简报结构、风险边界、拍板项 |
| 怎么沉淀学习 | 知识卡片、能力点、private-first 建议 |
| 怎么对比竞品 | 对比维度、差异点、可借鉴点 |

## 2. 业务推进方向

适合银行员工业务推进时，优先给：

1. 目标客户或目标场景
2. 当前最值得验证的问题
3. 可以低风险开启的话术
4. 后续推进动作
5. 不宜过早承诺的内容

## 3. 管理与汇报方向

适合基层管理和领导汇报时，优先给：

1. 事项背景
2. 关键变化
3. 影响对象
4. 当前风险或机会
5. 建议动作
6. 需要拍板或协调的事项

## 4. 能力沉淀方向

信息值得长期保留时，优先 private-first：

- 客户或个人经验 -> `workspace/private/case-notes`
- 用户偏好或常用打法 -> `workspace/private/memories`
- 可复用任务方法 -> `workspace/private/skills/action`
- 专业判断视角 -> `workspace/private/skills/reference`
- 公开事实且有来源 -> public `knowledge`

进入 public 前必须补来源、去敏感、补边界。
