# Signal Classification

## 1. 信息类型

先把输入分成一个主类型，最多保留一个副类型。

| 类型 | 典型输入 | 优先关注 |
|---|---|---|
| policy | 监管政策、政府文件、协会规则、监管口径 | 约束变化、鼓励方向、合规边界 |
| news | 媒体报道、机构动态、行业事件 | 事实可靠性、变化信号、影响对象 |
| product | 银行或金融机构产品、服务方案、办理说明 | 目标客户、场景、价值主张、边界 |
| competitor | 行外竞品、同业动作、互联网平台能力 | 对比维度、差异化、可借鉴打法 |
| client | 新客户、客户经营变化、融资需求、业务扩张 | 客户画像、痛点、触达切口 |
| market | 宏观、行业、利率、汇率、大宗商品 | 经营压力、现金流、风险偏好 |
| risk | 违约、舆情、经营异常、政策限制 | 风险传导、止损动作、上提机制 |
| capability | 值得学习的方法、表达、案例或工作范式 | 可复用能力、沉淀路径 |
| other | 无法明确归类的信息 | 先问用途，再做最小解读 |

## 2. 相关性判断

判断这条信息更接近哪个使用口径：

- 金融能力成长：重点解释“怎么看、怎么表达、怎么学习”
- 公开咨询：重点解释“公开事实是什么、能给外部用户怎么说”
- 银行员工业务推进：重点解释“客户、产品、场景、下一步动作”
- 基层管理与跟进：重点解释“谁跟、怎么拆、怎么检查”
- 领导汇报与拍板：重点解释“变化、影响、建议、拍板项”

## 3. 信息强度

按证据强度标注：

- high：来自官网、监管、上市公司公告、正式披露
- medium：来自可信媒体、机构研报、公开演讲或采访
- low：来自社交媒体、二手转述、未给来源的摘要
- unknown：只有用户描述，缺少来源

信息强度低时，仍可做工作启发，但必须明确“待核验”。
