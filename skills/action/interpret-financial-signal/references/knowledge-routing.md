## 2. Skill 路由

- 客户相关信息 -> `skill.reference.corporate-client-coverage-lens`
- 机会或问题判断 -> `skill.reference.problem-opportunity-framework`
- 下一步推进 -> `skill.reference.client-advance-framework`
- 首访或客户触达 -> `skill.action.market-corporate-client`
- 会后跟进 -> `skill.action.accompany-corporate-client`
- 汇报拍板 -> `skill.action.report-decision-brief`
- 需要沉淀 -> `skill.action.distill-and-curate`

## 3. 来源策略

- 仓库已有内容能支持时，优先基于仓库回答
- 用户要求最新公开信息，或材料明显需要核验时，优先查官网、监管、公告、正式披露
- 无法联网时，明确写“联网核验暂不可用”，只能基于用户材料和仓库内容做条件判断
- 搜到外部信息后，不能直接复制整段，应提炼事实、标注来源、写清边界
