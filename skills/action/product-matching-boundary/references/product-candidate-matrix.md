# 产品候选矩阵

## 使用说明

本矩阵根据客户需求类型生成“候选方向”，但所有具体产品名称必须来自：

`knowledge/banks/ceb/transaction-banking/product-catalog/products.md`

如果产品不在目录中，不能作为光大交易银行产品输出。
如果产品在目录中但详情状态为“待补充”，只能做方向性提示，不能编造功能、准入、费率、时效或办理条件。

## 禁止误输出

- 不得输出 `阳光e支付`。该名称不在当前光大交易银行产品目录中。
- 不得用历史知识包中的品牌、产品族或旧称替代当前产品目录。
- 不得把“供应链金融”泛泛需求直接等同于 `e付通`；需先判断是否明确涉及 1+N 保理、核心企业与供应商应收账款融资或线上化确权。

## 需求类型 -> 产品候选方向

### 1. 资金沉淀与现金管理

| 客户信号 | 可核验产品方向 | 备注 |
|---|---|---|
| 有阶段性闲置资金，希望提高收益 | 对公智能存款 | 详情待补充，不能承诺收益、期限或办理条件 |
| 多账户、多主体资金集中管理 | 集团资金池、现金管理云财资系统、现金管理跨行通系统 | 先确认集团架构、账户分布和跨行需求 |
| 跨境资金集中或境内外资金调拨 | 跨境资金池、跨境人民币汇款、跨境外币汇款 | 需核验外汇、跨境资金管理和真实贸易背景 |
| 收付款、对账、虚账户管理 | 实名制虚账簿、非实名制虚账簿、交易资金管理、现金管理协议支付 | 先判断是账户体系、支付流程还是资金监管问题 |

### 2. 基础结算与支付

| 客户信号 | 可核验产品方向 | 备注 |
|---|---|---|
| 日常境内收付款 | 境内人民币汇款、境内外币汇款 | 需区分币种、路径、账户和合规材料 |
| 跨境收付款 | 跨境人民币汇款、跨境外币汇款 | 需核验跨境结算背景 |
| 财务人员需要便捷支付或账户授权 | 单位结算卡、现金管理协议支付 | 单位结算卡已有部分公开资料，仍需核验客户权限管理要求 |
| 缴税、预付卡或支付机构资金场景 | 缴税业务、单用途商业预付卡、非银支付机构备付金存管 | 属特定场景，先确认客户身份和监管要求 |

### 3. 保函与信用支持

| 客户信号 | 可核验产品方向 | 备注 |
|---|---|---|
| 招投标、工程、履约、预付款等信用支持需求 | 保函 | 详情待补充，不能承诺出函条件 |
| 客户需要向交易对手证明融资能力或授信支持 | 信贷证明 | 详情待补充，需核验用途和文本要求 |

### 4. 汇率与结售汇

| 客户信号 | 可核验产品方向 | 备注 |
|---|---|---|
| 即期外汇收支结算 | 对公即期结售汇 | 需核验真实贸易背景和结售汇材料 |
| 锁定未来汇率 | 对公远期结售汇 | 需核验风险中性和套保需求 |
| 管理汇率波动或币种错配 | 代客人民币与外汇掉期、代客人民币与外汇期权 | 详情待补充，不能承诺结构、价格或适当性结论 |

### 5. 国内贸易与供应链

| 客户信号 | 可核验产品方向 | 备注 |
|---|---|---|
| 核心企业与经销商、供应商之间有货款、库存或订单协同 | 保兑仓、全程通、订单贷 | 先核验交易链条、货权、订单和回款来源 |
| 核心企业希望支持供应商应收账款融资 | 1+N无追索权保理、1＋N有追索权保理、e付通 | 只有明确涉及 1+N 保理、e付通或线上化确权时，才进入 e付通知识包 |
| 应收账款买断或转让融资 | 国内保理-无追索权、国内有追索权保理 | 需核验应收账款真实性、确权和回款路径 |
| 线上化销售、回款或经销商协同 | e收通、e销通 | 详情待补充，不能编造系统能力或接入条件 |

### 6. 信用证、托收与贸易融资

| 客户信号 | 可核验产品方向 | 备注 |
|---|---|---|
| 国内采购付款、账期安排或信用证结算 | 国内信用证、信用证（国内）融资-买方押汇、信用证（国内）融资-议付 | 国内信用证与议付已有部分公开资料，仍需核验业务条件 |
| 进口贸易付款或单据融资 | 进口信用证、进口代收、进口信用证押汇、进口代收押汇、汇出汇款押汇 | 需核验贸易背景、单据和付款责任 |
| 出口贸易收款或单据融资 | 出口信用证、出口托收、出口托收押汇、合格出口信用证押汇、合格出口信用证贴现、不合格出口信用证押汇、不合格出口信用证贴现、汇入汇款押汇 | 需区分信用证/托收/汇款路径及合格或不合格单据 |
| 出口备货或订单阶段融资 | 打包贷款、订单贷 | 需核验订单、信用证或合同基础 |
| 买断远期应收或无追索转让 | 国际福费廷、国内福费廷 | 详情待补充，需核验基础交易和票据/单据条件 |
| 跨境应收账款或贸易项下保理 | 国际保理 | 详情待补充，需核验交易结构和买方风险 |

## 输出格式

```text
候选方向：
- [产品名称]：为什么可能相关；当前详情状态；需要核验什么

不能直接下结论：
- 是否能办、额度、价格、时效、准入条件，均需进一步核验
```

## 多产品组合建议

当单一产品无法覆盖需求时，可以只给“组合方向”，不要直接承诺方案成立：

- 现金管理 + 汇款结算
- 信用证 + 贸易融资
- 保理 + 订单/回款核验
- 结售汇 + 跨境汇款

组合时仍需逐一核验产品目录、客户资质、真实交易背景和合规边界。
