# Knowledge Routing

## 光大产品路由

### 光大交易银行产品目录

- **命中条件**：涉及光大交易银行产品、产品推荐、产品比较、产品名核验或客户适配
- **知识包**：`pack.banks.ceb.transaction-banking.product-catalog`
- **适用场景**：先确认可输出的产品名称范围，避免使用仓库未覆盖或不在当前清单中的名称
- **硬边界**：不在产品目录里的名称不得作为光大交易银行产品输出；`阳光e支付` 不在目录中，不得推荐或解释为光大交易银行产品
- **平台边界**：`阳光供应链云平台` 是对客服务平台/线上化渠道能力，可作为承载平台说明；不能替代具体产品名做推荐

### 结算底座

- **命中条件**：客户卡点在账户、收付、回单、对账、付款流程、授权管理
- **知识包**：`pack.banks.ceb.corporate-settlement.basic-settlement`
- **适用场景**：基础结算需求；卡点在收付款执行层面而非融资层面
- **边界提示**：结算产品通常边界较宽，重点看企业是否在可服务范围内

### e付通（1+N 保理线上化场景）

- **命中条件**：明确出现 e付通、1+N 保理、核心企业授信支持供应商融资、应收账款转让、线上化确权或相关公开材料
- **知识包**：`pack.banks.ceb.transaction-banking.e-fu-tong`
- **适用场景**：客户处于核心企业-供应商交易链中，且已明确进入 e付通或 1+N 保理线上化讨论
- **行业触发**：行业本身不能触发 e付通；必须有核心企业、供应商、应收账款、1+N 保理或 e付通明确信号
- **边界提示**：需确认核心企业准入情况；供应链融资涉及贸易背景真实性核查

### 电费通（电费证）

- **命中条件**：出现电费、电费缴纳、电网、电费结算单、国内证、福费廷、电费支付优化、月度缴费
- **知识包**：`pack.banks.ceb.trade-finance.dian-fei-tong`
- **适用场景**：客户有持续、大额电费支出，卡点在缴费节奏与资金占用优化
- **行业触发**：高耗能制造（化工、钢铁、有色金属）、数据中心、大型园区、商业综合体
- **边界提示**：电费证业务涉及国内证开证条件；关注客户与电网的结算周期

### 供应链与保理方向

- **命中条件**：泛供应链金融、保理、供应链平台、核心企业上下游、应收账款融资
- **知识包**：先读 `pack.banks.ceb.transaction-banking.product-catalog`，需要解释平台背景时再读 `pack.banks.ceb.transaction-banking.yangguang-supply-chain`；具体产品按白名单选择 `国际保理`、`国内保理-无追索权`、`国内有追索权保理`、`1+N无追索权保理`、`1＋N有追索权保理`、`e付通`、`订单贷`、`e收通`、`e销通` 等待核验方向
- **适用场景**：需要先判断产品大类和待核验产品，不宜直接锁定单一产品
- **边界提示**：阳光供应链云平台只说明线上化承载能力；保函涉及银行或有负债；保理涉及应收账款确权

### 普惠场景金融

- **命中条件**：小微企业、个体工商户、涉农、灵工、物流、安居、数字政务场景
- **知识包**：`pack.banks.ceb.inclusive-finance.inclusive-finance-scenario-map`
- **适用场景**：普惠口径客户需要快速融资、场景嵌入、线上化审批
- **边界提示**：普惠贷款有额度和资质要求；场景金融涉及特定场景合规性

## 通用能力路由

- 客户需求诊断结论需进一步验证 -> `pack.common.sales.consultative-b2b`
- 客户处于犹豫或异议阶段 -> `pack.common.psychology.business-communication`

## 路由优先级

1. **产品目录优先**：涉及光大交易银行产品名时，先读产品目录白名单
2. **场景特异性优先**：产品知识包（结算 / e付通 / 电费通 / 供应链）优先于通用能力包
3. **产品知识包内**：按场景特异性选主包，不明确时按回退逻辑
4. **多包命中**：最多选 1 个产品主包 + 1 个产品备选包 + 1 个通用能力包

## 回退逻辑

- 命中供应链/核心企业/应收账款场景但未明确产品 -> 先走产品目录和需求诊断，只输出“供应链金融/保理方向待核验”，不默认走 e付通
- 命中电费/能源缴费场景但未明确产品 -> 进入电费通/电费证场景核验
- 命中基础收付/账户问题 -> 走基础结算
- 无法判断场景类型 -> 走基础结算，并在输出中标注"需进一步厘清客户站位"
- 命中小微/普惠但场景不明确 -> 走普惠场景地图

## 读取建议

1. 先读取产品主包 `README.md` 与 `modules/*`
2. 再读取产品主包 `faq.md`，用于边界快速确认
3. 如有备选产品包，补读其 `README.md` 和 `faq.md`
4. 最后补通用能力包，提升适配判断质量

## 多包场景处理

- 命中多个产品包时，按场景特异性选主包，另一个作为备选
- 输出时明确"主推方向"与"备选方向"
- 如需跨条线协同，明确标注并说明协同边界
