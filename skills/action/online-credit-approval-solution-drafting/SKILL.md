# 线上授信自动化审批业务方案辅助撰写 Skill

## Scope

这是一个 action skill。
它负责通过问答收集信息、识别缺口，并输出线上授信自动化审批业务方案的章节草稿，尤其聚焦"授信方案"部分。

## When To Use

- 分行需要起草线上授信自动化审批业务方案
- 已有部分材料，但不知道还缺什么
- 需要把"授信方案"章节先拉成可讨论草稿
- 需要把准入策略、额度、期限、用途、还款方式等内容先结构化

## Input Contract

### 最低输入

- 展业场景
- 目标客户群体
- 核心企业或合作平台基本情况
- 初步授信模式或还款来源

### 可选补充

- 历史方案模板
- 核心企业或平台股权、经营和财务材料
- 数据来源和核验能力说明
- 风险缓释安排
- 反欺诈、反洗钱、消保、贷后管理要求

## Checklist（补数清单）

### 1. 商业模式

- 展业场景
- 目标客户群体
- 授信方案初步思路
- 业务风险管控要点和措施

### 2. 核心企业 / 合作平台

- 概况、股权结构、经营和财务状况
- 与我行合作情况
- 整体业务规模、资产质量、同业合作情况

### 3. 客户群体

- 客户特征、引流渠道、还款能力分析
- 准入标准和条件

### 4. 数据情况

- 数据来源、真实性与可核验性
- 对风险识别的支撑能力

### 5. 风险缓释

- 担保方式、风险缓释抓手
- 对核心风险点的覆盖度

### 6. 授信管理

- 信息收集、贷前调查和信息核验
- 反欺诈管理、反洗钱管理
- 审批管理、放款审核、贷后管理

### 7. 消费者权益保护管理

- 咨询管理、投诉管理、合作方消保要求

### 8. 授信方案章节重点

- 审批模式、准入策略、贷款额度、授信期限
- 贷款期限、贷款用途、还款方式
- 风险缓释、场景价值评估

## Workflow

1. 先按 Checklist 收集最低输入
2. 信息不足时先输出缺口清单和追问问题，不强行写满
3. 重点先起草"授信方案"章节
4. 准入策略只能给结构化建议，不给最终政策口径
5. 结尾必须列出待人工确认项

## Output Contract

### 必须输出

1. **当前已掌握信息** — 只写已经明确提供或公开可确认的内容
2. **缺口清单** — 列出还需要补充的材料，不要含糊写"资料不足"
3. **需追问问题** — 优先追问会直接影响"授信方案"章节的点
4. **授信方案章节草稿** — 至少覆盖：审批模式、准入策略、贷款额度、授信期限、贷款期限、贷款用途、还款方式、风险缓释、场景价值评估
5. **准入策略建议草稿**
6. **待人工确认项** — 必须单独列出：内部政策口径、审批结论、定价与时效、关键阈值

### 输出语气

- 区分事实、建议、待确认
- 不写成拍板语气
- 不把建议冒充为结论

### 禁止在输出中

- 承诺审批通过
- 承诺额度、定价、期限最终口径
- 引用内部制度原文
- 编造核心企业、平台或客户数据

## Quality Gate

- 是否先问答补数，再起草
- 是否把"授信方案"作为中心输出
- 是否区分可草拟内容和待确认内容
- 是否明确边界和风险提示

## Examples

### 示例 1：资料不全时

推荐流程：先确认场景、客群、核心企业或平台 → 列出还缺的数据、风控、管理和消保材料 → 先草拟"授信方案"骨架 → 最后列待人工确认项。

### 示例 2：准入策略建议写法

"准入策略建议围绕交易真实性、核心企业或平台稳定性、还款来源可解释性、数据覆盖度和风险缓释安排分层设计，具体阈值待内部政策确认。"

### 示例 3：核心企业上下游场景

**输入**：围绕某制造业核心企业上游供应商的线上融资场景，目标客群为稳定合作的中小供应商，核心企业具备稳定采购和结算能力。

**输出要点**：
- 已掌握：场景属核心企业上下游供应链，借款主体为上游供应商，还款来源依赖贸易回款和经营现金流
- 缺口：核心企业对订单/对账/发票/回款的支持深度、供应商筛选规则、数据交叉核验能力、贷后监测安排
- 追问：核心企业能提供哪些可持续可核验数据、回款是否可归集、是否有反欺诈和贷后预警安排
- 授信方案草稿：审批模式建议保留异常样本人工复核；准入策略围绕核心企业稳定性、供应商合作基础、交易连续性分层设计；贷款用途限定为流动资金周转

### 示例 4：平台交易场景

**输入**：围绕某产业交易平台上的对公商户融资场景，目标客群为有稳定交易记录的企业商户，平台具备订单、结算、履约和经营数据。

**输出要点**：
- 已掌握：场景属平台交易驱动的对公线上融资，借款主体为平台内持续经营的企业商户
- 缺口：平台对交易真实性的支持深度、数据连续性和可交叉验证性、商户经营稳定性、消保安排
- 追问：平台能提供哪些关键数据、如何识别虚假交易和刷单、结算链条是否支撑回款监测
- 授信方案草稿：审批模式建议总行集中管理下线上化审批；准入策略围绕平台交易真实性、商户经营稳定性、数据可核验能力分层设计；场景价值评估从商户覆盖广度、交易频率、数据沉淀程度等维度说明
