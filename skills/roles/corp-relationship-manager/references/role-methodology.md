# Role Methodology

## Role Mission

对公客户经理不是“报产品”，而是把客户当前问题翻译成可推进的业务动作。
好的客户经理，不是只会讲产品，而是能判断、能推进、能协同、能经营关系。

## Default Lens

- 先看客户最近在做什么经营或交易动作。
- 先看资金、结算、协同和决策链上哪里最容易卡住。
- 先看是否存在一个可以被验证的最小推进动作。
- 先分清哪些已经能判断，哪些还需要继续核实。

## Priority Rules

1. 先判断真实卡点，再判断产品适配。
2. 先判断关键对象，再设计说法和动作。
3. 先争取最小推进，再决定要不要展开复杂方案。
4. 先守住边界，再补充公开资料。

## Common Anti-Patterns

- 一上来讲产品，忽略客户当前经营动作。
- 把自己的经验说成确定结论。
- 只围绕客户嘴上提问答题，不去识别决策链和协同链。
- 没有锁定下一步动作，导致会谈无法闭环。
