# 场景矩阵与产品匹配

## 按贸易类型匹配

| 场景 | 推荐产品 | 说明 |
|------|----------|------|
| 国内贸易、应收账款融资 | 阳光保理 | 聚焦应收账款管理和融资 |
| 国内贸易、核心企业-供应商链 | e付通（已有知识包） | 1+N 保理线上化模式 |
| 国内贸易、电费缴纳场景 | 电费通（已有知识包） | 国内证+福费廷组合 |
| 跨境贸易、进出口结算 | 阳光国际结算 | 信用证、托收、汇款 |
| 自贸区企业跨境业务 | 阳光自贸通 | 自贸区综合金融服务 |
| 企业资产估值融资 | 阳光资财 | 资产估值与融资 |
| 需要银行信用担保 | 阳光保函 | 投标、履约、预付款等保函 |
| 供应链全流程线上化 | 阳光供应链云平台 | 数字化供应链管理 |

---

## 按企业角色匹配

### 核心企业

- 需要管理供应商融资 → 阳光保理 / e付通
- 需要供应链数字化管理 → 阳光供应链云平台
- 需要跨境采购结算 → 阳光国际结算

### 供应商

- 对核心企业有应收账款 → 阳光保理 / e付通
- 需要应收账款融资 → 阳光保理
- 电费缴纳场景 → 电费通

### 贸易企业

- 进出口贸易结算 → 阳光国际结算
- 自贸区跨境业务 → 阳光自贸通
- 需要贸易担保 → 阳光保函

### 制造企业

- 原材料采购融资 → 阳光资财
- 供应商管理 → 阳光供应链云平台
- 电费管理 → 电费通

---

## 首次沟通判断要点

1. **贸易类型**：国内还是跨境？→ 决定保理/国际结算方向
2. **企业角色**：核心企业还是供应商？→ 决定融资模式
3. **线上化需求**：是否需要全流程线上化？→ 决定是否推荐云平台
4. **已有系统**：是否有 ERP/财务系统对接需求？→ 影响产品选择
5. **担保需求**：是否需要银行保函？→ 补充阳光保函

## 常见误区

- 把阳光供应链等同于单一产品（实际是产品族）
- 忽视客户实际贸易场景，直接推荐复杂产品
- 把 e付通和电费通当作与阳光供应链无关的独立产品
- 认为所有供应链产品都适合所有企业
