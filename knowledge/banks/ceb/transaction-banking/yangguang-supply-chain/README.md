# 光大银行-阳光供应链知识包

## 当前定位

本包用于解释光大交易银行供应链相关品牌、平台和场景背景，不是当前产品白名单。

涉及具体产品推荐、产品比较或产品适配时，必须先读 `knowledge/banks/ceb/transaction-banking/product-catalog/products.md`。其中：

- `阳光供应链云平台` 可以作为对客服务平台/线上化渠道能力说明。
- 未进入产品目录的品牌或旧称，不能作为当前光大交易银行产品直接推荐。
- 具体融资、结算、单证、保理产品名称，以产品目录为准。

## 适用问题

- 阳光供应链相关平台和场景背景是什么
- 阳光供应链云平台作为对客服务平台，适合承载哪些供应链线上化场景
- e付通和电费通与阳光供应链是什么关系
- 首次沟通供应链金融业务应先判断什么

## 模块

- `product-family.md` — 阳光供应链产品族总览与各子产品定位
- `scenarios.md` — 场景矩阵与产品匹配
- `existing-packs.md` — 与已有 e付通、电费通知识包的关联

## 使用建议（给 Agent）

1. 先读产品目录，确认哪些名称可以作为具体产品输出。
2. 再读 `product-family`，理解阳光供应链相关平台和场景背景。
3. 再读 `scenarios`，根据客户场景判断方向，但具体产品名称仍需回到产品目录。
4. 需要了解已有知识包时读 `existing-packs`，获取 e付通和电费通的详细信息路径。
5. 客户问答阶段优先读本文件的「常见问题」章节，保持结论稳妥。
6. 如果需要把知识转成会谈提纲或推进动作，补读 `skills/action/market-corporate-client`。

## 边界

该包只支持公开知识解释与产品体系理解，不输出审批、授信、费率、时效、系统对接能力或上线承诺。

## 常见问题

### Q1: 阳光供应链是一个产品还是一组产品？

**A:** 阳光供应链更适合作为供应链金融相关品牌、平台和场景背景理解，不应直接当成一个可推荐的单一产品。当前对外输出具体光大交易银行产品时，应先回到 `product-catalog/products.md`；其中阳光供应链云平台可以作为对客服务平台/线上化渠道能力说明。

---

### Q2: e付通和阳光供应链是什么关系？

**A:** e付通是供应链场景下的具体产品之一，适用于核心企业-供应商、1+N 保理、应收账款转让和线上化协同等场景。阳光供应链云平台可以作为相关线上化承载能力说明，但输出产品名时仍应写 `e付通` 或产品目录中的其他具体产品名。

---

### Q3: 阳光保理和 e付通有什么区别？

**A:** 当前输出时不要把 `阳光保理` 作为具体产品名直接推荐。需要表达保理方向时，应回到产品目录中的 `国内保理-无追索权`、`国内有追索权保理`、`国际保理`、`1+N无追索权保理`、`1＋N有追索权保理` 或 `e付通` 等名称，再根据客户场景核验适配。

---

### Q4: 企业应该选择哪个供应链产品？

**A:** 取决于企业的实际需求：
- 有核心企业-供应商融资需求 → 1+N 保理、国内保理、e付通等待核验
- 有电费缴纳融资需求 → 电费通
- 需要跨境贸易结算 → 跨境汇款、托收、信用证等待核验
- 是自贸区企业 → 跨境资金池、跨境汇款、结售汇等待核验
- 需要银行担保 → 保函
- 需要供应链数字化管理 → 阳光供应链云平台作为平台/渠道能力说明

建议首次沟通时先了解客户的贸易类型、企业角色和线上化需求。

---

### Q5: 阳光供应链云平台和 e付通的线上化有什么区别？

**A:** 阳光供应链云平台是光大银行的供应链线上化对客服务平台/渠道能力，偏向承载和协同。e付通是产品目录中的具体产品名，偏向核心企业-供应商、1+N 保理和应收账款融资场景。回答客户时不要用平台名替代具体产品名。

---

### Q6: 自贸区企业怎么判断方向？

**A:** 先判断企业是否有跨境收付、跨境资金集中、结售汇、信用证、托收等实际需求，再回到产品目录中的 `跨境资金池`、`跨境人民币汇款`、`跨境外币汇款`、`对公即期结售汇`、`对公远期结售汇` 等名称做核验。不要直接输出目录未覆盖的旧称。

---

### Q7: 供应链产品的费率是多少？

**A:** 公开资料未披露阳光供应链各子产品的具体费率、定价标准或收费方式。费率属于银行内部定价，需要与银行客户经理沟通确认。
