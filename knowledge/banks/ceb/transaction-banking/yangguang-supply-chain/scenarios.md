# 场景矩阵与产品方向

## 按贸易类型匹配

以下矩阵用于判断供应链场景方向。落到具体光大交易银行产品名时，必须回到 `product-catalog/products.md`。

| 场景 | 产品方向或平台能力 | 说明 |
|------|----------|------|
| 国内贸易、应收账款融资 | 国内保理、国际保理、1+N 保理、e付通等待核验 | 聚焦应收账款管理和融资 |
| 国内贸易、核心企业-供应商链 | e付通（已有知识包） | 1+N 保理线上化模式 |
| 国内贸易、电费缴纳场景 | 电费通（已有知识包） | 国内证+福费廷组合 |
| 跨境贸易、进出口结算 | 跨境汇款、托收、信用证等待核验 | 以产品目录内名称表达 |
| 自贸区企业跨境业务 | 跨境资金池、跨境汇款、结售汇等待核验 | 先核验客户场景和产品目录 |
| 企业资产估值融资 | 当前产品目录未覆盖 | 需要补充来源和产品边界 |
| 需要银行信用担保 | 保函 | 投标、履约、预付款等保函方向 |
| 供应链全流程线上化 | 阳光供应链云平台 | 对客服务平台/线上化渠道能力，不替代具体产品名 |

## 按企业角色匹配

### 核心企业

- 需要管理供应商融资 → 1+N 保理、国内保理、e付通等待核验
- 需要供应链数字化管理 → 阳光供应链云平台
- 需要跨境采购结算 → 跨境汇款、托收、信用证等待核验

### 供应商

- 对核心企业有应收账款 → 1+N 保理、国内保理、e付通等待核验
- 需要应收账款融资 → 国内保理、国际保理等待核验
- 电费缴纳场景 → 电费通

### 贸易企业

- 进出口贸易结算 → 跨境汇款、托收、信用证等待核验
- 自贸区跨境业务 → 跨境资金池、跨境汇款、结售汇等待核验
- 需要贸易担保 → 保函

### 制造企业

- 原材料采购融资 → 订单贷、保兑仓、全程通等待核验
- 供应商管理 → 阳光供应链云平台
- 电费管理 → 电费通

## 首次沟通判断要点

1. **贸易类型**：国内还是跨境？→ 决定保理/国际结算方向
2. **企业角色**：核心企业还是供应商？→ 决定融资模式
3. **线上化需求**：是否需要全流程线上化？→ 决定是否需要说明阳光供应链云平台
4. **已有系统**：是否有 ERP/财务系统对接需求？→ 影响产品选择
5. **担保需求**：是否需要银行保函？→ 补充保函方向

## 常见误区

- 把阳光供应链等同于单一产品（实际是产品族）
- 忽视客户实际贸易场景，直接推荐复杂产品
- 把 e付通和电费通当作与阳光供应链无关的独立产品
- 认为所有供应链产品都适合所有企业
- 把阳光供应链云平台当作具体融资产品推荐
