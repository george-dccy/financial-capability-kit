# 光大银行-电费通（电费证）知识包

## 适用问题

- 电费通（电费证）在公开资料里更接近什么场景方案
- 电费缴纳为什么会和国内证、福费廷放在一起理解
- 首次讨论前该先整理哪些信息，哪些结论不能先给

## 命名规则

本目录路径中的 `dian-fei-tong` 只是文件 slug，不是产品英文名。
对用户输出时只能写 `电费通` 或 `电费证`，不得翻译或改写成其他目录外名称。

## 模块

- `scenario-positioning.md`
- `preparation-checklist.md`
- `customer-faq.md`

## 使用建议（给 Agent）

1. 先读 `scenario-positioning`，确认客户是否真的处在"按月缴电费"的稳定场景里。
2. 再读 `preparation-checklist`，把账单、发票、缴费节奏和参与角色拆清楚。
3. 客户答疑阶段优先读 `customer-faq` 与本文件的「常见问题」章节，把"场景理解"与"审批结论"严格分开。
4. 如果需要把知识转成客户推进动作，补读 `skills/action/market-corporate-client`。

## 边界

该包只用于公开信息解释与场景理解，不构成审批、额度、费率、时效和授信承诺。

## 常见问题

### 可以直接判断是否适配该方案吗

不可以。该包不输出具体准入与审批判断。

### 可以直接给额度、费率与时效吗

不可以。即使公开手册里出现过局部示例或参数，本仓库也不把它写成对外承诺。

### 首次会谈最值得先问什么

先问三件事：  
电费支付是否稳定发生、账单/发票基础是否清楚、这次沟通要先达成什么最小结果。

### 更多常见问答

详见 `customer-faq.md`。
