# 光大银行公开基础知识包

## 定位

这个 knowledge pack 只沉淀公开、稳定、可引用、适合外部传播的内容。

它服务的不是正式办理，而是前期理解、问题分流、材料准备和沟通提效。

## 当前收录范围

- `modules/corp-settlement.md`：对公基础结算
- `modules/e-payment.md`：阳光 e 付通
- `modules/electricity-certificate.md`：阳光电费证
- `faq.md`：常见问题与边界口径
- `sources.md`：来源索引

## 使用建议

- 当问题是“这是什么”“适合什么场景”“前期要准备什么”时，优先使用这个 pack
- 当问题变成“怎么推进客户”“怎么组织汇报”“怎么持续跟进”时，应切回 role 或 workflow
- 当问题涉及额度、定价、审批、时效或正式受理结论时，必须明确提示以实际沟通结果为准

## 编写原则

- 只写公开知识，不写内部口径
- 尽量用业务语言解释，不堆砌产品术语
- 每个模块都优先回答：是什么、什么时候会问到、前期怎么理解、哪些边界不能越

## 当前版本边界

当前版本只做“公开基础知识包”，不追求覆盖全部产品，不承担官方问答入口角色。
